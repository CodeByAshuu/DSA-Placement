#                                           - - - INHERITANCE - - -

Inheritance allows one class (derived class) to acquire properties and behaviors of another class (base class).
- Promotes code reusability
- Supports polymorphism
- Implements IS-A relationship

‚ùóEffect of visibility mode

protected	    protected	        protected	            private

‚úÖCONTRUCTOR DESTRUCTOR ORDER
Order of execution:
- Base constructor runs first
- Then derived constructor
- On destruction ‚Üí reverse order
ex: 
    Base()
    Derived()
    ~Derived()
    ~Base()


‚úÖ Diamond Problem
Two copies of base class is copied - ambiguity
solution ? virtual inheritance using 'virtual' keyword


‚úÖ OBJECT SLICING
When a derived class object is assigned to a base class variable , Extra derived data is sliced off:
Base b = Derived();


‚úÖ Important MCQ points:
- Constructors are not inherited
- Destructors are not inherited
- Copy constructor not inherited
- Assignment operator not inherited

‚úÖ FUNCTION OVERRIDING
Same function name + same signature in base and derived class.
- Achieved using virtual function for runtime polymorphism
- Late binding occurs

Rule:
virtual void show();
void show() override;


‚úÖ COMPOSITION vs INHERITANCE

Inheritance = IS-A
Composition = HAS-A (preferred)
Important interview question:
Use inheritance when behavior is shared, not just data.


‚úÖ Friendship is not inherited.

‚úÖ STATIC MEMBER IN INHERITANCE
- Inherited
- Shared across all objects
- Accessed via derived class too

‚úÖ BASE INHERITED USING PROTECTED
If derived class inherits base using protected mode:
Public ‚Üí becomes protected
Protected ‚Üí stays protected

‚úÖ CONSTRUCTOR IN MULTIPLE INHERITANCE

Order follows class order in declaration, not initialization list.
class D : public B, public C {};
Order:
B()
C()
D()

‚úÖ FINAL KEYWORD
Stops inheritance or overriding.
class A final { };
virtual void show() final;
